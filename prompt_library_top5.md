# Prompt Library (Top 5 우선순위 + 전체 15종 템플릿)
버전: 1.0  
용도: 개발 서버/문서 시스템에 바로 반영 가능한 Markdown 단일 파일

---

## 0) 사용 방법(요약)
- 문서 내 프롬프트는 **복사-붙여넣기**로 바로 사용 가능하도록 설계했습니다.
- `{{변수}}`만 상황에 맞게 치환하십시오.
- **Top 5**는 실무 사용 빈도가 높고, 자동화/운영(예: n8n, 내부 도구)로 확장하기 쉬운 형태로 상세화했습니다.
- 나머지 10개는 “기본 템플릿”으로 제공됩니다.

---

## 1) 공통 변수(Variable) 사전
- `{{주제}}`: 다루려는 주제(키워드 단위로 구체화 권장)
- `{{타겟}}`: 독자/고객/사용자(직군, 연차, 지역 등 포함)
- `{{목표}}`: 목적(인지/유입/전환/리텐션/운영 효율 등)
- `{{기간}}`: 분석/캘린더/리서치 범위(예: 최근 30일, 다음 4주)
- `{{채널}}`: 플랫폼/매체(예: Instagram, TikTok, Google Search 등)
- `{{KPI}}`: 성공지표(예: CTR, CVR, CAC, ROAS, NPS, Retention)
- `{{제약사항}}`: 예산/인력/툴/브랜드 규정/컴플라이언스
- `{{입력데이터}}`: 표/로그/리포트/설문 원문 등

# 2) Top 5 (우선순위) — 상세 템플릿
아래 5개는 **실무 사용성 + 자동화 확장성** 기준으로 우선순위로 선정했습니다.

## Top 1) 마케팅 전략 — 소셜 콘텐츠 캘린더(1개월) (ID: T4)
### 프롬프트(복붙용)
```text
당신은 {{업종}}의 시니어 디지털 마케팅 전략가입니다.
브랜드/제품: {{브랜드}}
기간: {{다음 1개월}}
목표: {{인지/유입/리드/구매 전환/재구매}}
타겟: {{타겟 페르소나(연령/직군/관심사/지역)}} 
핵심 오퍼/메시지: {{오퍼/USP}}
채널: {{Instagram, TikTok, YouTube, LinkedIn, Naver, Kakao 등}}
제약사항: {{브랜드 가이드/금지표현/예산/리소스/운영 빈도}}

산출물 요구사항:
1) 주차별(Week1~Week4/5) 콘텐츠 캘린더를 표로 작성
2) 각 포스트 필드:
   - 플랫폼, 날짜(또는 요일), 포맷(릴스/캐러셀/숏폼/이미지/라이브 등)
   - 주제(한 줄), 훅(한 줄), 핵심 메시지(2~3줄), CTA(한 줄)
   - 제작 가이드(핵심 소재/촬영 포인트/디자인 포인트)
   - 추천 해시태그(플랫폼별 10개 이내)
3) 플랫폼별 ‘최적 게시 시간’ 제안(근거: 일반 패턴 + 가정 명시)
4) 운영 체크리스트(제작→검수→게시→리포트) + 담당 역할 분장(RACI 간단 버전)
5) KPI 설계: 채널별 1~2개 핵심 KPI 및 측정 방법 제안

출력 형식(반드시 준수):
- (A) 캘린더 표 1개
- (B) 플랫폼별 운영 가이드(불릿)
- (C) KPI/측정 계획 표 1개
- (D) 실행 체크리스트 1개
```
### 출력 스키마(자동화/서버 반영용)
- 캘린더 표 컬럼 권장
  - `week`, `date_or_day`, `platform`, `format`, `theme`, `hook`, `key_message`, `cta`, `production_notes`, `hashtags`, `kpi`
- KPI 표 컬럼 권장
  - `platform`, `primary_kpi`, `secondary_kpi`, `definition`, `measurement_tool`, `reporting_frequency`

### 품질 체크(수용 기준)
- 각 주차에 **플랫폼별 최소 2개 이상** 콘텐츠가 배치되어 있는가
- 메시지/CTA가 목표(인지/전환 등)와 일치하는가
- 해시태그는 과다(10개 초과) 또는 중복이 심하지 않은가
- KPI가 “측정 가능”하고(도구/정의 명확) 운영 주기(주간/월간)가 제시됐는가

### 미니 예시(변수 치환 샘플)
- 브랜드: A 패션 커머스
- 목표: 신규 카테고리 런칭 유입 + 장바구니 전환
- 채널: Instagram, TikTok, Naver Blog, Kakao
- KPI: LP CTR, ATC Rate, CVR

## Top 2) 콘텐츠 제작 — 실행형 블로그 아티클(SEO/AEO/GEO) (ID: T1)
### 프롬프트(복붙용)
```text
당신은 {{분야}}의 시니어 콘텐츠 에디터이자 퍼포먼스 마케터입니다.
주제: {{주제}}
목표 독자: {{타겟}}
목표: {{검색 유입/리드/브랜드 신뢰/설득}}
콘텐츠 채널: {{블로그(네이버/워드프레스)/뉴스레터/링크드인 등}}
지역(GEO): {{지역 키워드(예: 서울/경기/부산/한국)}} 
핵심 키워드(SEO): {{키워드 8~12개}}
제약사항: {{브랜드 가이드/금지 표현/근거 필요 여부}}

작성 요구사항:
- 분량: {{2000~3000자 또는 1000 words}}
- 구성(순서 고정):
  1) 제목(SEO 고려)
  2) 도입부(문제/공감 → 약속)
  3) Executive Summary(5줄)
  4) 본문 5개 섹션(H2) + 각 섹션에 실무 예시 1개
  5) FAQ(AEO): 독자가 묻는 질문 5개 + 짧고 직접적인 답변
  6) 결론: 실행 체크리스트 7개 + 다음 액션(CTA)
- 문체: {{전문적/설명형/설득형}}, 문단 구분 명확
- 반드시 포함:
  - 숫자/지표/체크리스트(최소 2개)
  - 불확실한 내용은 “가정”으로 명시

출력 형식:
- Markdown
- H2/H3 구조 사용
- 마지막에 해시태그 10개 이내
```
### 서버 반영용 메타(선택)
- YAML front matter를 쓰는 시스템이라면 상단에 아래를 추가해도 됩니다.
```yaml
---
title: "{{SEO 제목}}"
category: "{{카테고리}}"
tags: [{{tag1}}, {{tag2}}, {{tag3}}]
created_at: "{{YYYY-MM-DD}}"
---
```

### 품질 체크(수용 기준)
- 도입부가 “독자 문제”를 직접 언급하는가
- FAQ 답변이 장황하지 않고 **직접적인 결론형**인가
- 체크리스트/표가 최소 2개 포함되는가
- 키워드가 억지로 반복되지 않고 자연스럽게 포함되는가

## Top 3) 데이터 분석 및 시각화 — 인사이트+액션 플랜 (ID: T3)
### 프롬프트(복붙용)
```text
당신은 {{업종}}의 데이터 애널리스트입니다.
아래 데이터셋을 분석해, 실행 가능한 인사이트와 시각화 계획을 제안하세요.

[DATA]
{{입력데이터(표/CSV/로그/요약)}}
[/DATA]

분석 목적: {{목적(원인 파악/성과 개선/이상탐지/예측)}}
핵심 질문: {{핵심 질문 3개}}
제약사항: {{데이터 한계/기간/표본/비식별/툴}}

산출물 요구사항:
1) Executive Summary(5줄)
2) Key Findings(최소 7개): 각각 근거(수치/패턴) 포함
3) 이상치/결측/중복 등 데이터 품질 진단 및 처리 권고
4) 시각화 계획 6종:
   - 차트 타입, 축 정의, 목적(무엇을 보여주는지), 주의점
5) 액션 플랜 5개:
   - “무엇을/왜/어떻게/측정(KPI)” 형태로 작성
6) 추가 가설/추가 데이터 요청 3개(필수)

출력 형식:
- 섹션 헤더 고정
- 표는 Markdown 테이블로 작성
```
### 시각화 추천 세트(기본)
- 추세: 라인(기간 vs KPI)
- 분포: 히스토그램/박스플롯
- 비교: 막대(카테고리 vs KPI)
- 상관: 산점도(단, 인과 아님을 명시)
- 퍼널: 단계별 전환율
- 코호트: 리텐션 매트릭스

### 품질 체크(수용 기준)
- “발견”이 “액션”으로 연결되는가(각 finding에 action 최소 1개 매핑 권장)
- 데이터 품질 이슈가 누락되지 않았는가
- 시각화가 목적과 축이 명확한가

## Top 4) 프로젝트 관리 — 리스크 관리 계획(실행형) (ID: T10)
### 프롬프트(복붙용)
```text
당신은 {{프로젝트}}의 PM입니다.
프로젝트 리스크 관리 계획을 작성하세요.

프로젝트명: {{프로젝트}}
기간: {{기간}}
목표: {{목표}}
범위(In/Out): {{범위}}
제약사항: {{예산/인력/일정/보안/법무/PII 금지 등}}

산출물 요구사항:
1) 리스크 레지스터 15개 이상(분류 포함: 일정/기술/인력/품질/보안/법무/외부 의존성)
2) 각 리스크마다:
   - 발생확률(H/M/L), 영향도(H/M/L), 탐지 시그널(조기경보), 대응전략(회피/완화/전가/수용)
   - Owner(담당), Due(기한), 상태(Status)
3) Top 5 리스크는 상세 대응 플랜(작업 단위, 우선순위, 커뮤니케이션 플랜)
4) 운영 방식: 주간 리스크 리뷰 아젠다 + 업데이트 규칙(변경관리)

출력 형식:
- (A) 리스크 레지스터 표
- (B) Top 5 상세 대응(소제목)
- (C) 운영 규칙 체크리스트
```
### 품질 체크(수용 기준)
- 리스크가 “나열”에 그치지 않고 Owner/Due가 들어갔는가
- 조기경보 시그널이 “측정 가능”한 형태인가
- 운영 규칙(리뷰 주기, 변경관리)이 명문화되어 있는가

## Top 5) 운영 최적화 — 프로세스 개선(As-Is → To-Be) (ID: T13)
### 프롬프트(복붙용)
```text
당신은 운영 혁신(Operational Excellence) 컨설턴트입니다.
아래 프로세스를 As-Is로 정리하고, 병목을 찾아 To-Be 개선안을 제안하세요.

대상 프로세스: {{프로세스(예: 콘텐츠 제작→검수→발행→리포팅)}}
현재 문제: {{문제(지연/오류/재작업/비용)}}
목표: {{리드타임 단축/오류율 감소/생산성 향상}}
제약사항: {{툴/인력/권한/컴플라이언스}}

산출물 요구사항:
1) As-Is 단계 10~20 스텝(입력/출력/담당 포함)
2) 병목/낭비 지점 7개(근거: 대기/핸드오프/재작업/승인 지연 등)
3) To-Be 프로세스(단계 재설계) + 자동화 포인트 5개
4) 표준화 문서/체크리스트 제안(최소 2종)
5) KPI 설계(리드타임, 오류율, 재작업률, 처리량 등) + 측정 방법

출력 형식:
- (A) As-Is 표
- (B) 병목 리스트
- (C) To-Be 표
- (D) KPI 표
```
### 품질 체크(수용 기준)
- As-Is에 “담당/입력/출력”이 빠지지 않았는가
- 개선안이 실제 제약(인력/권한/툴) 내에서 실행 가능한가
- KPI가 정의+측정방법까지 포함되는가

# 3) 전체 15종 템플릿(기본 라이브러리)
아래는 이미지에 있던 15개 항목을 모두 포함한 기본 템플릿입니다(Top 5는 위에서 상세화).

## 1) 콘텐츠 제작(Content Creation) (ID: T1)
```text
당신은 {{분야}}의 시니어 콘텐츠 에디터입니다.
주제: {{주제}} / 타겟: {{타겟}} / 목적: {{목적}}

요구사항:
- 분량: {{분량}}
- 구성: Executive Summary + 5개 섹션(H2) + 결론(실행 Takeaways)
- 톤: {{톤}}
- 포함: 키워드 {{키워드}}, 예시 2개, 체크리스트 1개

출력: Markdown
```

## 2) 소프트웨어 개발(Software Development) (ID: T2)
```text
Python으로 {{기능}}을 구현하는 클래스를 작성하세요.
- 메서드 정의, docstring, 인라인 주석 포함
- 타입 힌트 및 예외 처리 포함
- 사용 예시 포함
출력: 코드만
```

## 3) 데이터 분석 및 시각화(Data Analysis & Visualization) (ID: T3)
```text
[DATA]{{데이터}}[/DATA]
- 핵심 요약, 트렌드, 시각화 기법 추천, 추가 가설을 제안하세요.
출력: 요약/발견/시각화/다음 액션 구조
```

## 4) 마케팅 전략(Marketing Strategy) (ID: T4)
```text
브랜드: {{브랜드}} / 기간: {{1개월}} / 채널: {{채널}}
- 콘텐츠 캘린더, 게시 아이디어, 최적 게시 시간, 해시태그를 제안하세요.
출력: 표 + 운영 가이드
```

## 5) 비즈니스 커뮤니케이션(Business Communication) (ID: T5)
```text
수신자: {{수신자}} / 주제: {{주제}} / 목적: {{요청/보고/설득}}
- 명확하고 간결한 이메일을 작성하세요.
출력: Subject + 본문
```

## 6) 전략 기획(Strategic Planning) (ID: T6)
```text
대상: {{회사/제품}}
- SWOT를 각 항목 5개 이상 작성하고, 전략을 제안하세요.
출력: SWOT 표 + 전략
```

## 7) 재무 모델링(Financial Modeling) (ID: T7)
```text
비즈니스: {{유형}}
- 5개년 재무모델의 구조, 필수 지표, 가정을 정리하세요.
출력: 구조도 + KPI/가정 목록
```

## 8) 인사(Human Resources) (ID: T8)
```text
직무: {{직무명}}
- JD를 작성하세요(책임/자격요건/우대/회사 소개 포함).
출력: JD 문서
```

## 9) 법무 문서(Legal Document Preparation) (ID: T9)
```text
NDA 범위: {{사업 영역}}
- 핵심 조항과 중요성을 개요로 작성하세요(법률 자문 대체 아님).
```
